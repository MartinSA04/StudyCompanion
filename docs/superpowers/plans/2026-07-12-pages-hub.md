# GitHub Pages Course Hub Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A one-pager at `kurs.martinsundal.no` (GitHub Pages, this repo) with a
tile per course site, grouped by semester, in the framework's exact visual
language.

**Architecture:** A second, tiny Astro build (`astro.config.hub.mjs`,
`srcDir: hub/`, no integration) whose single page imports the framework's real
`src/styles/*` files and reuses `ThemeToggle.astro` + `faviconDataUri()` — zero
copied design code. Tiles come from `hub/courses.yaml`. A dedicated GitHub
Actions workflow deploys `dist-hub/` to Pages.

**Tech Stack:** Astro 6 (already a devDep), `yaml` (already a dep), node:test,
Playwright (existing `visual/` setup), GitHub Actions Pages deploy.

**Spec:** `docs/superpowers/specs/2026-07-12-pages-hub-design.md`

## Global Constraints

- Norwegian copy, verbatim from the spec: headline `Studieguider`, intro
  "Interaktive studieguider med flashcards, quiz og formelsamling."
- Framework styles are IMPORTED (`src/styles/fonts.css`, `tokens.css`,
  `base.css`, `primitives.css`) — never copied.
- Theme storage key: `sc:theme:hub` (framework's `sc:theme:<code>` pattern).
- GoatCounter endpoint: `https://kursmartinsundal.goatcounter.com/count`,
  injected in production builds only (`import.meta.env.PROD`), `is:inline`.
- Custom domain: `kurs.martinsundal.no` via `hub/public/CNAME`.
- Term codes match `^[HV]\d{4}$` (framework convention, e.g. `H2026`).
- No per-course logic in the framework (`src/index.ts`, schema untouched).
- Format every new/edited file with `npx prettier --write <files>` before
  committing (repo style; several hub files fall outside the `lint` globs).
- Commit messages: conventional commits, ending with
  `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
- The repo may have a stale `pnpm preview` server on port 4321 (it serves
  `dist/` statically). Before any `test:visual` run, rebuild first so it
  serves fresh output: `pnpm build && pnpm hub:build`.

```ts
  assert.ok(courses.length > 0);

test("every course has code, title, https url and a framework term code", () => {

    assert.ok(c.code?.trim(), `code missing: ${JSON.stringify(c)}`);
    assert.ok(c.title?.trim(), `title missing in ${c.code}`);

    assert.match(c.term ?? "", /^[HV]\d{4}$/, `bad term in ${c.code}`);
  }
});

test("course codes are unique", () => {
  const codes = courses.map((c) => c.code);
```

- [ ] **Step 2: Add the no-flash script + ClientRouter to the head**

Directly before `</head>`, add:

```astro
    <script is:inline define:vars={{ themeKey }}>
      // No-flash theme init — trimmed from CourseLayout (single page: no
      // view-transition swaps to re-apply across). An explicit stored choice
      // wins; otherwise the OS preference does, tracked live. The OS path
      // dispatches sc:themechange so ThemeToggle can mirror aria-pressed.
      (function () {
        function isDark() {
          try {
            const saved = localStorage.getItem(themeKey);
            return saved
              ? saved === "dark"
              : matchMedia("(prefers-color-scheme: dark)").matches;
          } catch (e) {
            return false;
          }
        }
        function applyTheme() {
          if (isDark())
            document.documentElement.setAttribute("data-theme", "dark");
          else document.documentElement.removeAttribute("data-theme");
        }
        applyTheme();
        matchMedia("(prefers-color-scheme: dark)").addEventListener(
          "change",
          function () {
            try {
              if (localStorage.getItem(themeKey)) return;
            } catch (e) {
              /* storage disabled — OS still wins */
            }
            applyTheme();
            document.dispatchEvent(
              new CustomEvent("sc:themechange", {
                detail:
                  document.documentElement.getAttribute("data-theme") ===
                  "dark"
                    ? "dark"
                    : "light",
              }),
            );
          },
        );
      })();
    </script>
    {/* ClientRouter so astro:page-load fires and ThemeToggle's onPage wiring
        runs — the same lifecycle contract the toggle has in CourseLayout. */}
    <ClientRouter />
```

```yaml
name: Deploy hub to GitHub Pages

on:
  push:
    branches: [main]
    paths:
      - "hub/**"
      - "src/styles/**"
      - "src/components/ThemeToggle.astro"
      - "src/components/Icon.astro"
      - "src/lib/favicon.ts"

  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      # --no-frozen-lockfile: mirrors visual.yml (CI-only devDeps may lag the
      # committed lockfile).
      - uses: actions/setup-node@v4

          node-version: 22
          cache: pnpm
      - run: pnpm install --no-frozen-lockfile
      - run: pnpm hub:build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist-hub/

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```
